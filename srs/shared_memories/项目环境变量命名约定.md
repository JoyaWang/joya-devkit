# 项目环境变量命名约定

## 规则：dev 环境带 `_DEV` 后缀，prd 环境无后缀

当同一变量需要区分 dev/prd 环境时：
- **dev 环境**：变量名加 `_DEV` 后缀
- **prd 环境**：变量名无后缀（默认/生产态）

## 示例（InfoV SRS 接入）

| 含义 | dev 变量名 | prd 变量名 |
|------|-----------|-----------|
| SRS Service Token | `SRS_TOKEN_INFOV_DEV` | `SRS_TOKEN_INFOV` |
| SRS Base URL | `SRS_BASE_URL_DEV` | `SRS_BASE_URL` |
| SRS API Base URL | `SRS_API_BASE_URL_DEV` | `SRS_API_BASE_URL` |
| SRS Service Token (Server) | `SRS_SERVICE_TOKEN_DEV` | `SRS_SERVICE_TOKEN` |

## 为什么这样设计

- prd 是默认/生产态，不应有额外后缀，保持最简
- dev 是特例/开发态，用 `_DEV` 明确标识
- 避免 `SRS_TOKEN_INFOV_PRD` 这种冗余表达（prd 本身就是默认）

## 适用范围

- GitHub Actions secrets
- GitHub Actions variables
- 服务器 .env 文件
- docker-compose.yml 环境变量注入
