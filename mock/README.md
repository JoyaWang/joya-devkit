# mock/

`mock/` 用于承载跨阶段共享的模拟数据与模拟实现，服务于原型、开发、测试和演示。

## 目录说明

### `fixtures/`
共享 JSON 数据文件。

用途：
- 原型阶段展示真实感数据
- 开发阶段复用同一批样例数据
- 演示阶段保持与原型一致的数据语义

原则：
- 不在 bootstrap 阶段预填虚假占位业务数据
- 真实 fixture 在原型或接口契约明确后再创建

### `factories/`
数据生成逻辑。

用途：
- 批量生成对象、版本、审计日志、测试记录
- 构造边界场景、异常场景和压力样本

### `repositories/`
Mock 接口实现。

用途：
- 在真实共享服务尚未接通前提供稳定接口
- 在测试和 demo 环境中替代真实依赖

## 跨阶段共享原则

- 同一份 fixture 尽量贯穿 **原型 → 开发 → 测试 → 演示**
- 业务代码依赖接口，不依赖具体 mock 文件
- mock 和 real 的切换由配置或依赖注入决定，不在业务逻辑中硬编码
- 即使未来接入真实共享服务，mock 层也保留用于测试、演示和故障回退验证
