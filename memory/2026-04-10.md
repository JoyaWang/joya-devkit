# 2026-04-10

## shared storage read fallback 执行层
- 本轮把 shared storage 读路径从“只取首候选 binding”升级到“最多两候选顺序尝试”：先 active primary location 对应 binding，再 resolver 当前 binding（若不同）。
- 新增统一 helper：先 `factory.getOrCreate(binding)`，再 `adapter.headObject({ objectKey })`；只有 `exists === true` 才 `createDownloadRequest()`，否则继续下一候选。
- `download-requests.ts` 与 `public-delivery.ts` 现已复用同一 helper；两候选都不存在时返回 404。
- `public-stable` 在 `download-requests.ts` 中继续返回稳定公共 URL，没有退化成 provider 临时签名 URL。
- 测试层经验：当读路径统一先走 `headObject` 后，历史用例里的 adapter mock 也必须补全 `headObject`，否则会出现“测试夹具不满足真实 contract”的假失败。
