# 前端设计规范

## 设计原则
本项目当前 Phase 1 只建设共享运行时 API 与 worker，不建设独立终端前端。因此本文件只保留控制面接入约束，供 `admin-platform` 或后续控制台页面接入时参考。

## 色彩体系
Phase 1 暂不定义独立视觉系统，由控制面宿主项目继承其既有设计系统。

## 排版
沿用控制面宿主项目的字体、字号和排版规则；本项目自身不定义单独品牌排版。

## 组件规范
如后续需要为 release / object 管理建立独立控制台：
- 列表页优先采用表格 + 筛选栏布局
- 详情页优先采用摘要卡片 + 审计时间线
- 风险操作（删除对象、切换 rollout、强更）必须二次确认

## 布局
- 运营查询类页面：筛选条件区 + 结果列表区 + 详情抽屉 / 明细页
- 不在 Phase 1 引入复杂自定义 dashboard

## 图标和图形
沿用宿主控制台的图标体系。

## 动效
- 避免不必要动效
- 风险操作和异步处理中优先强调状态清晰与审计可追溯

## 接入约束
- 控制面只做可视化与操作入口，不承担共享运行时真相源
- 所有控制台操作都必须通过 shared-runtime-services API，而不是直接跳过服务层操作底层资源
